# 全局搜索契约

状态：产品语义事实源。

分析页全局搜索同时召回筛选候选项和 item 字段匹配。两类召回相互独立：筛选候选项只有在用户勾选后才产生过滤效果，字段匹配则随当前查询直接产生结果。

## 筛选候选项

全局搜索只在分析页侧边栏已有的以下选项中查找候选项：

- 年份；
- 保险库；
- Domain。

候选项列表与侧边栏使用同一数据源和同一选择状态。全局搜索不生成凭据类型候选项，也不搜索密码、TOTP 或 Passkey 等凭据类型。

输入查询只控制当前展示哪些候选项，不改变已有选择。候选项必须由用户显式勾选；修改或清空查询不能取消已经选择的筛选条件。在全局搜索或侧边栏任意一处切换选项，另一处必须同步反映。

## 结构化筛选集合 A

每个筛选组内的选项取并集，不同筛选组之间取交集。所有集合运算都以 item 为单位：

```text
yearItems = items(year1) union items(year2) ...
vaultItems = items(vault1) union items(vault2) ...
domainItems = items(domain1) union items(domain2) ...

A = selected category sets intersection
```

没有选择任何选项的筛选组不参与交集。年份、保险库和 Domain 都没有选择项时，`A` 为空集，而不是全体 item。

同一个 item 必须满足所有参与交集的筛选组。不能由同一相似组内的不同 item 分别满足不同条件。

## 字段搜索集合 B

当前查询同时搜索已经进入相似分析结果的 item。搜索范围仅包括：

- title；
- username；
- email；
- URL 原始文本。

年份、保险库、Domain、凭据类型、电话和备注不属于字段搜索范围。Domain 通过筛选候选项召回，URL 仍按完整原始文本参与字段搜索。

查询执行 NFKC 规范化、忽略大小写并按空白拆分关键词。所有关键词必须在同一个 item 内全部匹配；不同关键词可以命中该 item 的不同可搜索字段。

## 最终结果

当存在结构化选择或非空查询时，最终 item 结果为：

```text
Result = A union B
```

没有结构化选择且查询为空时显示全部相似组。凭据类型仍可通过侧边栏独立筛选，但不出现在全局候选项中。

最终展示以相似组为单位：只要组内至少一个 item 属于 `Result`，就展示完整相似组，供用户判断保留、归档或删除。筛选结果同时限定批量执行计划的作用域。
